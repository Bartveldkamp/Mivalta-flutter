**Why Mac-only:** the Android `.so` is proven by `smoke` in CI, but the iOS
binary (`.xcframework`) can only be built on macOS with Xcode. It must be rebuilt
at the new pin so iOS also links SQLCipher-with-vendored-OpenSSL.

1. `git pull` on `main` (has the `3db58f2` pin).
2. Build the iOS artifact:
   ```bash
   cd /path/to/Mivalta-flutter
   scripts/build_ios.sh
   ```
   - **No FRB regen:** `engine_registry.json` is **v2.24 unchanged** across the
     bump (zero method delta), so the shim (`rust/src/api.rs`) and `lib/src/rust/`
     bindings are untouched — `frb-drift-guard` stays satisfied, `lib/src/rust/`
     git-clean.
   - **Expected:** SQLCipher compiles for the iOS target with vendored OpenSSL
     (same fix as Android, now exercised on the Apple toolchain). Build finishes
     clean.
   - **If OpenSSL needs build deps** (perl/make) they're present in a standard
     Xcode CLT environment; install if missing — do **not** revert the feature.
3. **On-sim sanity:** `flutter run` → readiness home renders as before (the pin
   bump is behaviour-preserving for display; vault runtime unchanged).
